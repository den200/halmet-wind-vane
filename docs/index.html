⛵ A boat project

The wind display died.
The vane up the mast didn't.

One day the old ST60+ wind display at the helm just went dark. I assumed the whole instrument was gone — but it turned out only the screen had died. The sensor at the top of the mast was still up there, still spinning, with nothing left to listen to it. This is how I got it talking again.

The whole thing is on GitHub →

Only the screen died

A Raymarine ST60+ is really two boxes sharing one cable: a sensor up the mast, and a display at the helm that knows how to read it. The display is the part that gave up. The sensor — a little vane for wind direction, three cups for wind speed — was completely fine. Fifteen metres up, perfectly happy, and suddenly completely deaf, because the only thing that ever understood it was the dead screen.

Meanwhile the rest of the boat had moved on. The Orca and everything else talk to each other over NMEA 2000, a network the 90s-era ST60+ never spoke a word of. So nothing was actually broken except a screen. The job wasn't to repair anything — it was to translate: read the old sensor's signals directly, and put the wind back onto the network everything else already uses.

It just needed something new to listen to it.

How do you read a wind vane built for a 1990s display?

This was the part I wasn't sure about. It turns out the vane doesn't send its angle as a number — it sends two voltages. One rises and falls with the sine of the angle, the other with the cosine. A bit like how the length and the direction of a sundial's shadow together tell you the time: either one alone is ambiguous, but together they pin down exactly where the vane is pointing. So you measure both with a precise voltmeter and compare them, and the angle falls out.

Speed is the easy half. The cups click a switch as they spin — faster wind, faster clicks — so you just count the clicks over half a second. The board does both of these ten times a second, then wraps the result up in the standard wind message the rest of the boat already understands.

One small thing I'm glad I built in: if the sensor ever goes quiet, the board says "no reading" instead of freezing on the last angle it saw. A wind needle stuck at a confident, wrong number is worse than an honest blank.

The build

The brains is a Hat Labs HALMET — a small board that's happy with the rough voltages on a boat, reads the sensor with a precise 16-bit voltmeter chip, and plugs straight into the NMEA 2000 backbone, drawing its own power from it. Three wires from the existing mast cable go in: sine, cosine, and the speed pulse. That's the whole connection.

The sensor wants 8 volts, which the board doesn't hand out, so that comes from a small adjustable power supply. This is the one place it's easy to ruin everything:

⚠ Set the power supply to 8.0 V before anything else. These cheap adjustable supplies arrive set to around 20 V — more than enough to cook the sensor. Dial it down to 8.0 V with a meter, with nothing connected to the output, and only then let the red wire anywhere near it.

Tuning it by sailing

Once it was wired, none of the fine-tuning needed a laptop. The board hosts its own little web page, so everything you'd want to adjust — which way is forward, which direction it rotates, the speed scale — is just a number you type into a browser. I set the zero by sighting down the boat, and flipped the rotation with a single setting when it first read backwards. The angle I dialled in by letting the vane spin freely and logging the readings; the speed I checked against GPS on a calm day.

It only ever reports apparent wind — the wind you actually feel on deck. I deliberately left "true wind" for the Orca to work out, so there's one honest source and nothing on the network fighting over the same number. The same readings also go out over WiFi to SignalK, so the tablet and any SignalK gauge can show them too.

And the part that does the actual sending is about as boring as it should be:

// Every 100 ms: package the latest calibrated wind reading and send it.
event_loop()->onRepeat(100, []() {
  tN2kMsg msg;
  double awa = isfinite(last_awa_rad)
      ? (last_awa_rad < 0 ? last_awa_rad + 2*M_PI : last_awa_rad)  // −π..π → 0..2π
      : N2kDoubleNA;
  double aws = isfinite(last_aws_mps) ? last_aws_mps : N2kDoubleNA;
  SetN2kWindSpeed(msg, sid, aws, awa, N2kWind_Apparent);
  nmea2000->SendMsg(msg);   // → onto the backbone
});

Where it ended up

It works. Direction and speed are back on the Orca, the same numbers land in SignalK over WiFi for the tablet, and the whole thing updates ten times a second — from a sensor everyone had already written off. I tested it on the bench first, then watched it come alive properly the first time a gust hit the masthead on the water.

The display that died is in a drawer somewhere. The sensor it left deaf is doing exactly what it always did. It just needed someone new to listen.

Under the hood it's an ESP32 running the SensESP framework, the NMEA2000 library doing the network side, and an ADS1115 doing the careful measuring — all open source. If you want the wiring, the calibration notes, or the whole firmware, it's on GitHub.
